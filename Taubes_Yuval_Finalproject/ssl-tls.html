SSL/TLS

What is SSL and when was it replaced?

Secure Socket Layer (SSL) is the predecessor to Transport Layer Security (TLS). SSL was released in 1995 and was replaced by TLS in 1999. The most recent version of TLS is RFC 8446, which was released in August of 2018. The most common use for SSL/TLS is secure web browsing. Secure browsing is also known as Hypertext transfer secure (HTTPS). The easiest way of telling if a website is using SSL/TLS is the lock to the left of the URL/search bar.

The Lock determines whether a website is using HTTPS (using SSL/TLS), or HTTP.

A website using HTTP instead of HTTPS.

How does SSL/TLS encrypt HTTPS?

TLS creates two keys, one private and one public. The Public key is shared with the user’s PC and the private key is kept by the website’s server. The keys are used to encrypt and decrypt data from both the website’s server and the user’s PC. However, when a user opens a connection a new set of keys are created, called session keys. Session keys are used to further encrypt the data between the website’s server and the user’s PC. By using these keys all data transferred between the connection is seen as a random string of characters, by hackers

What is the difference between HTTP and HTTPS?

HTTP is a protocol that allows 2 computers to communicate over the internet using 2 kinds of messages, requests, and responses. HTTP requests and responses are sent over the internet using plaintext (unencrypted information). The issue with this is that hackers can see the plaintext sent over the internet, making HTTP unsecure . HTTPS uses TLS which means the information that you send and receive is encrypted (not written in plaintext).
